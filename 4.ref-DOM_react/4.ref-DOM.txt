JSX 안에서 id를 사용을 권장 하지않고 ref를 권장한다.
ref는 전역적으로 작동하지 않고 컴포넌트 내부에서만 작동하기 때문에 id와 같은기능이지만 편리하다.

-ref를 사용해아할 상황
DOM을 직접적으로 건드려아 할 때 사용한다. 
* 특정 input에 포커스 주기
* 스크롤 박스 조작
*Canvas 요소에 그림 그리기 등

-ref 사용법
ref를 달아야하는 DOM에 props를 설정하듯이 하면 된다. ref 값으로는 콜백 함수를 전달 골백 함수는 ref를 파라미터로 가지며
콜백 함수 내부에서 컴포넌트의 멤버 변수에 ref를 담는 코드를 작성하면 끝.
ex) <input ref={(ref) => {this.input = ref}}/>
이렇게 작성하면 this.input은 input요소의 DOM을 가리킨다. ref의 이름은 DOM타입과 관계없이 아무렇게나 작성 가능.
this.aaa = ref

-컴포넌트에 ref 달기
리엑트에서는 컴포넌트에서도 ref를 달수 있다. 이방법은 주로 컴포넌트 내부에있는 DOM을 컴포넌트 외부에서 사용할 때 사용.
ex) ref={(ref) => {
        this.myComponent = ref
        }   
    }

- ref사용의 잘못된 예
서로 다른 컴포넌트끼리 데이터를 주고 받을 때 ref를 사용해도 되지만 리엑트 사상의 어긋난 설계이다.
앱 규모가 커지면 마치 스파게티처럼 구조가 꼬여버려서 유지보스가 불가능.
컴포넌트끼리 데이터를 교류할 때는 언제나 데이터를 부모 <-> 자식 흐름으로 교류해야 한다.
